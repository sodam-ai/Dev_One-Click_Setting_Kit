Dev_One-Click_Setting_Kit — 바이브코딩 환경 원클릭 설정 키트

Windows PC에서 AI로 코딩(“바이브코딩”)을 시작하는 데 필요한 개발 도구들을 파일 하나 더블클릭으로 자동 설치해 주는 도구입니다. 컴퓨터·AI를 처음 다루는 분도 그대로 따라 하면 됩니다.

1. 이게 뭔가요?

“바이브코딩”은 AI에게 한국어로 말하듯 요청해서 프로그램을 만드는 방식입니다. 그런데 AI 코딩을 하려면 먼저 PC에 여러 개발 도구(Git, Python, Node.js 등)를 깔아야 합니다. 하나씩 찾아 설치하면 처음엔 30분~1시간이 걸리고 실수도 잦습니다.

이 키트는 dev-one-click-setting-kit.bat 파일 하나만 실행하면:

  • 필요한 도구를 자동으로 찾아 설치하고,
  • 이미 있는 도구는 건너뛰거나 최신으로 올려주고,
  • 설치가 끝나면 결과 요약 파일을 만들어 줍니다.

⚠️ Windows 전용입니다. macOS·Linux에서는 작동하지 않습니다.

용어 풀이 (처음이면 이것만 알면 됩니다)

용어 아주 쉽게
터미널 / 콘솔 명령을 글자로 입력하는 검은 화면
winget Windows에 기본 내장된 “프로그램 자동 설치기”
CLI 마우스 대신 글자 명령으로 쓰는 프로그램
LTS “오래 지원되는 안정 버전”(장기 안전판)

2. 사전 준비물

항목 필요 조건
운영체제 Windows 10 (21H2, 빌드 19044 이상) 또는 Windows 11
인터넷 필수 (winget이 인터넷에서 도구를 내려받습니다)
디스크 여유 공간 왕초보 3GB / 전체 설치 시 약 7GB 이상 권장
권한 관리자 권한 불필요(대부분). 일부 도구만 실패 시 관리자 실행
winget Windows 10/11에 보통 내장. 없으면 설치 안내 참고

Windows 10 오래된 버전(21H1 이하)은 winget이 불안정합니다. 이 경우 키트가 경고하고 계속 여부를 물어봅니다.

3. 필요 프로그램(설치되는 도구 목록)

설치는 4단계 레벨(누적)로 나뉩니다. 상위 레벨은 하위 레벨을 모두 포함합니다.

레벨 누적 개수 추가되는 도구 예상 시간
왕초보 6 Git · Python 3 · Node.js LTS · VS Code · Windows Terminal · scoop ~7분
중급 12 + Git LFS · GitHub CLI · PowerShell 7 · pnpm · Bun · Ollama ~15분
고급 17 + Java 21 LTS · Go · Rust · Flutter+Dart · Stripe CLI ~35분
올인원 19 + Ruby · PHP ~45분

추가로 고를 수 있는 것 ([5] 선택 설치)

  • winget 도구: uv(Python 패키지 관리)
  • npm 도구(Y/N 선택): Vercel CLI · Supabase CLI · Stripe SDK · Resend SDK · Railway CLI · Clerk · Prisma · Uploadthing

특정 버전 설치 ([V] 버전 선택 설치) — 안정/LTS 버전만

  • Python: 3.11 / 3.12 / 3.13
  • Java(JDK): 8 / 11 / 17 / 21 (전부 LTS)
  • Ruby: 3.2 / 3.3

scoop — 왕초보 [1]~[4]에 자동 포함 · [S]로 단독 설치

  • scoop: 개발용 명령줄 패키지 매니저. winget에 없어 공식 설치 스크립트(get.scoop.sh)로 설치합니다. 레벨 [1]~[4] 설치에 자동 포함(레벨 시작 시 Y 확인 후)되고, [S]로 단독 설치도 됩니다. 보안 상세는 §13.

설치 후 자동으로 되는 것 (Post-install)

  • Git 줄바꿈 설정(core.autocrlf true) 자동
  • Git 이름·이메일은 자동 저장하지 않고 안내만 (개인정보 보호)
  • Python pip 최신화, Node의 광고 메시지 끄기(npm config set fund false)
  • Claude Code CLI 자동 설치 (성공/실패를 정직하게 표시)
  • Ollama 로컬 AI 모델은 안내만(자동 다운로드 안 함 — 용량 큼)
  • 설치된 도구가 잘 인식되는지 PATH 검증 출력

AI 에디터(Cursor, Claude Desktop 등)는 프로그램이라 자동 설치하지 않고 [8] 직접 다운로드에서 공식 페이지를 열어 줍니다.

4. 다운로드 방법

방법 A — Releases(권장) 저장소 페이지 오른쪽 Releases → 최신 버전 → Assets의 dev-one-click-setting-kit.bat 클릭해 저장.

방법 B — ZIP < > Code 버튼 → Download ZIP → 압축 해제 → 폴더 안의 dev-one-click-setting-kit.bat 사용.

파일 하나(.bat)만 있으면 됩니다. 다른 파일은 설명서·개발 기록입니다.

5. 빠른 시작 (3분 요약)

  1. dev-one-click-setting-kit.bat를 더블클릭.
  2. 파란 경고창(SmartScreen)이 뜨면 [추가 정보] → [실행].
  3. 메뉴가 나오면 처음이면 A(가장 쉬운 추천 설치) 또는 1(왕초보) 입력 후 Enter.
  4. Y로 설치 시작 → 이미 설치된 것 처리엔 그냥 Enter(건너뜀).
  5. 끝나면 안내대로 새 터미널을 열어 사용 시작.

6. 실행 방법

  1. 파일 받기 → 4. 다운로드.
  2. 더블클릭 실행. 관리자 권한은 대부분 불필요. 일부 실패 시 파일 우클릭 → “관리자 권한으로 실행”.
  3. SmartScreen 파란 창이 떠도 정상입니다: [추가 정보] → [실행].
  4. 검은 화면(터미널)에 메뉴가 뜹니다. 원하는 번호/문자를 입력하고 Enter.

인터넷으로 내려받기 때문에 시간이 걸릴 수 있습니다. 창을 닫지 마세요.

7. 사용 방법 (메뉴 설명)

실행하면 나오는 메인 메뉴:

[1] 왕초보 설치     처음 시작 (6개,  ~7분)
[2] 중급 설치       (12개, ~15분)
[3] 고급 설치       (17개, ~35분)
[4] 올인원 설치     (19개, ~45분)
[5] 선택 설치       원하는 번호만 골라서
[6] 업데이트        설치된 도구를 안정(LTS) 버전으로
[7] 제거            도구 삭제 (개별/전체)
[8] 직접 다운로드   공식 사이트 URL 목록 (winget 불가 시)
[9] 설치 확인       O/X + 버전 표시
[0] 종료
[A] 가장 쉬운 추천 설치   처음이면 이거! (기본 5종 + AI 안내)
[V] 버전 선택 설치        특정 안정버전 (Python/Java/Ruby)
[S] scoop 설치          개발용 패키지 매니저 (winget 외)
메뉴 설명
[1]~[4] 레벨별 일괄 설치. 설치 전 목록·소요시간을 보여주고 Y/N 확인
[A] 기본 5종 설치 + Cursor·Claude Desktop 다운로드 페이지 열기
[5] 번호를 쉼표로 입력(예: 1,3,8). Enter만 누르면 권장 기본팩(1~5). 고른 도구 이름을 미리 보여준 뒤 설치
[V] Python/Java/Ruby의 특정 안정 버전 선택. 잘 모르면 그냥 Enter(권장값)
[S] scoop(개발용 패키지 매니저) 설치. winget에 없어 공식 스크립트 사용 — 실행 전 동의(Y/N). 보안 상세 §13
[6] 이 키트가 설치한 도구만 안정(LTS) 버전으로 업데이트
[7] 개별/전체 제거
[8] winget으로 안 될 때 공식 다운로드 페이지를 브라우저로 엶 (URL은 Ctrl+클릭)
[9] 각 도구 설치 여부(O/X)와 버전 확인

8. 작동 방법 & 명령어

이 키트는 메뉴 번호만 입력하면 되므로 외울 명령어가 없습니다. 설치가 끝난 뒤, 새 터미널에서 아래로 잘 깔렸는지 확인할 수 있습니다.

git --version
python --version
node --version
claude --version

Git 첫 사용 시(1회만) — 이름·이메일 설정(키트는 안내만 하고 자동 저장하지 않음):

git config --global user.name  "홍길동"
git config --global user.email "your@email.com"

11. 워크플로우

더블클릭 실행
  → 첫 실행 환영/안내 (처음 1회)
  → 메뉴 선택
  → [사전 체크] Windows 버전 · winget · 패키지목록 갱신 · 인터넷 · 디스크 · 기존 설치 감지
  → 도구 설치 (의존성 순서: Git → Python·Node → pnpm·Bun → Java → Flutter …)
      · 이미 있으면 건너뜀/업그레이드 선택
      · 실패 시 5초 후 1회 자동 재시도 → 그래도 실패면 건너뜀([8] 안내)
  → 설치 후 자동 설정 (Git 줄바꿈, pip 최신화 등)
  → 리포트 2종 저장 + PATH 검증 출력
  → “새 터미널을 열어 시작하세요” 안내

12. 아키텍처

  • 단일 파일: 순수 Windows 배치 스크립트(.bat) 1개. 별도 설치/빌드 없음.
  • 패키지 관리: winget(Microsoft 공식·서명 검증) 사용. 각 도구는 공식 소스에서 설치.
  • 구조(서브루틴): 메뉴 → :PRE_CHECK(사전 점검) → :INSTALL/:NPM_INSTALL(설치·재시도) → :POST_*(설치 후 설정) → :MAKE_REPORTS(리포트) → :PATH_CHECK(검증).
  • 이식성: %~dp0 기반이라 파일 위치·이름을 바꿔도 작동.
  • 인코딩: CP949(한국어 Windows 기준). 영어 Windows에서는 메뉴 글자가 깨질 수 있음(알려진 제약 — 14 참고).

13. 보안 / 데이터 흐름

  • 관리자 권한 불필요(사용자 범위 설치). 기업/학교 PC 대응.
  • 공식 소스만(winget): winget 설치는 전부 --source winget(Microsoft 서명 검증).
  • 원격 스크립트 예외 — scoop 1건: scoop은 winget에 없어 공식 설치 스크립트(https://get.scoop.sh, HTTPS)를 실행합니다. 유일한 원격 실행 경로이며, 레벨 [1]~[4] 설치(시작 시 Y 확인) 또는 [S] 선택 시에만 실행됩니다(winget과 달리 서명 검증은 없고, 공식 출처만 사용).
  • 입력 최소화: 사용자는 메뉴 번호만 입력. 임의 명령 실행 통로 없음.
  • 개인정보 보호: Git 이름/이메일은 자동 저장하지 않고 안내만.
  • 외부 전송 없음: 만들어지는 파일은 로컬(리포트/로그)뿐. 서버로 보내지 않음.
  • 리포트 내용: PC 이름(COMPUTERNAME)·설치 결과만. 사용자명·홈 경로·비밀번호 등 민감정보 미포함.
  • 시크릿 없음: 코드/문서에 API 키·토큰·비밀번호가 들어있지 않습니다.

15. 자주 묻는 질문 (FAQ)

Q. 코딩을 한 번도 안 해봤는데 써도 되나요? 네. [A] 또는 [1]만 눌러도 기본 환경이 갖춰집니다.

Q. 다시 실행해도 되나요? 네. 이미 설치된 것은 기본값 건너뜀이라 안전합니다.

Q. 관리자 권한이 꼭 필요한가요? 대부분 아니요. 일부 도구만 실패할 때 관리자 실행을 안내합니다.

Q. 특정 버전(예: Python 3.11)을 깔 수 있나요? 네. [V] 버전 선택 설치에서 안정 버전만 고를 수 있습니다.

Q. 설치한 걸 지우려면? [7] 제거 → 개별 또는 전체.

Q. 인터넷이 꼭 필요한가요? 네. winget이 인터넷에서 도구를 내려받습니다.

Q. macOS/Linux에서 되나요? 아니요. Windows 전용입니다.
