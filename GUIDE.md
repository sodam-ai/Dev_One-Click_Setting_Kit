## 1. 이게 무엇인가요?

**AI에게 한국어로 말해서 프로그램을 만드는 "바이브코딩"** 을 시작하려면, PC에 여러 개발 도구(Git, Python 등)를 미리 깔아야 합니다. 하나씩 찾아 설치하면 30분~1시간이 걸리고 실수도 잦습니다.

**`dev-one-click-setting-kit.bat` 파일 하나만 실행하면**, 필요한 도구들을 **자동으로** 찾아 설치해 줍니다.

- 이미 깔린 도구는 **건너뛰기** 합니다 (다시 깔지 않음).
- 설치 결과를 **리포트 파일**로 저장합니다.
- **Windows 전용**입니다. macOS·Linux에서는 작동하지 않습니다.

## 2. 사전 준비물 (시작 전 확인)

| 항목 | 필요 조건 |
|------|-----------|
| 운영체제 | **Windows 10 (21H2 이상)** 또는 **Windows 11** |
| 인터넷 | **필수** (도구를 인터넷에서 내려받습니다) |
| 디스크 여유 공간 | 최소 **3GB** (전체 설치 시 약 5GB 권장) |
| winget | Windows에 기본 내장된 "앱 설치 관리자". 없으면 5번·12번 참고 |

> 관리자 권한은 **없어도 대부분 설치됩니다.** 일부 도구가 실패하면 관리자 권한으로 다시 실행하세요(5번 참고).

## 3. 빠른 시작 — 3단계

1. **파일 받기**: GitHub 페이지에서 `dev-one-click-setting-kit.bat` 다운로드 (4번 참고).
2. **실행**: 받은 파일을 **더블클릭**.
3. **선택**: 검은 화면(터미널)에 메뉴가 뜨면 **`A`** (가장 쉬운 추천 설치) 또는 **`1`** (초보자 설치)을 입력하고 **Enter**.

> 처음이라면 **`A`** 를 추천합니다. 기본 도구 + AI 코딩 도구 안내까지 한 번에 챙겨줍니다.

## 4. 다운로드 방법

**방법 1 — Releases (권장)**
1. GitHub 저장소 페이지 오른쪽의 **[Releases](../../releases/latest)** 클릭.
2. **Assets** 항목에서 `dev-one-click-setting-kit.bat` 클릭 → 저장.

**방법 2 — 전체 ZIP**
1. 초록색 **`<> Code`** 버튼 → **`Download ZIP`**.
2. 받은 ZIP을 **압축 해제** → 폴더 안의 `dev-one-click-setting-kit.bat` 사용.

> 저장 위치는 **바탕화면**이나 **다운로드** 폴더처럼 찾기 쉬운 곳을 추천합니다.

## 5. 실행 방법 (파란 경고창 대처 포함)

1. `dev-one-click-setting-kit.bat` 를 **더블클릭** 합니다.
2. (선택) 더 확실히 하려면 파일에서 **마우스 오른쪽 클릭 → "관리자 권한으로 실행"**.
3. **파란 경고창**("Windows의 PC 보호" / SmartScreen)이 떠도 **정상**입니다.
   - **[추가 정보]** 를 클릭 → **[실행]** 버튼을 누르면 됩니다.
4. 처음 실행하면 **환영 안내 화면**이 한 번 나옵니다(다음부터는 안 나옵니다). 아무 키나 누르면 메뉴로 넘어갑니다.

> ⚠️ 백신(Windows Defender 등)이 `.bat` 실행을 막을 수 있습니다. 그럴 땐 해당 파일을 허용 목록에 추가하거나 잠시 실시간 보호를 끄고 실행하세요.

## 6. 메뉴 사용 방법 (전체 설명)

메뉴에서 **번호(또는 글자)를 입력하고 Enter** 를 누릅니다.

| 입력 | 이름 | 설명 |
|------|------|------|
| **A** | 가장 쉬운 추천 설치 | **처음이면 이것!** 기본 5종 설치 + Cursor·Claude Desktop 다운로드 페이지를 열어줌 |
| **1** | 초보자 설치 | 기본 5종 (약 7분) |
| **2** | 중급 설치 | 초보자 포함 11종 (약 15분) |
| **3** | 고급 설치 | 중급 포함 16종 (약 35분) |
| **4** | 새로운 설치 | 고급 포함 18종 (약 45분) |
| **5** | 선택 설치 | 원하는 도구만 번호로 골라서 (쉼표로 여러 개: 예 `1,3,6`) |
| **6** | 안전 업데이트 | 설치된 도구를 **안정(LTS) 버전**으로 점검·업데이트 (실행 전 목록 확인) |
| **7** | 제거 | 개별 또는 전체 제거 |
| **8** | 직접 다운로드 | winget으로 안 될 때, 공식 사이트 24개 URL을 열어줌 |
| **9** | 설치 확인 | 무엇이 깔렸는지 O/X + 버전 표시 |
| **0** | 종료 | 프로그램 닫기 |

**설치 시작 전 "이미 설치된 도구 처리 방법"** 을 물어봅니다:
`[1] 건너뜀(기본값) / [2] 업그레이드 / [3] 제거` — 그냥 **Enter** 를 누르면 안전하게 **건너뜀**이 선택됩니다.

> 화면에서 결과는 색으로 구분됩니다: **완료=초록**, **실패=빨강**, 메뉴 제목=청록 (Windows 10 이상).

## 7. 설치되는 도구 목록

> 상위 레벨은 하위 레벨을 모두 포함합니다(누적).

**초보자 (5종)** — Git, Python 3, Node.js LTS, VS Code, Windows Terminal
**중급 (+6 = 11종)** — Git LFS, GitHub CLI, PowerShell 7, pnpm, Bun, Ollama
**고급 (+5 = 16종)** — Java 21 (LTS), Flutter, Go, Rust, Stripe CLI
**새로운 (+2 = 18종)** — Ruby, PHP

**선택 설치 추가 항목** — uv(Python 패키지 관리), 그리고 npm 도구(Vercel·Supabase·Stripe·Resend·Railway·Clerk·Prisma·Uploadthing).

**자동 함께 설치** — Claude Code CLI(터미널용 AI 코딩 도구), pip 최신화, npm 설정.

**프로그램(앱)은 자동 설치하지 않고 `[8] 직접 다운로드`로 안내** — Cursor, Claude Desktop, GitHub Desktop, LM Studio, Windsurf, Warp. (사용자가 직접 설치하도록. 올인원에 휩쓸려 깔리지 않습니다.)

## 8. 내부 작동 방법 (무슨 일이 일어나나)

설치를 고르면 순서대로 진행됩니다:

1. **사전 체크** — Windows 버전 / winget 유무 / 패키지 목록 갱신 / 인터넷 / **디스크 3GB** / 관리자 권한 안내 / 기존 Node·Python 감지.
2. **설치** — 의존성 순서대로(예: pnpm은 Node 다음, Flutter는 Java 다음). 각 도구는 `winget` 공식 소스에서 설치하고, **실패하면 5초 뒤 1회 자동 재시도** 후 그래도 안 되면 건너뜁니다(전체 중단 안 함).
3. **설치 후 자동 설정** — Git 줄바꿈 설정, pip 최신화, npm 광고 끄기, Claude Code CLI 설치 등.
4. **리포트 생성** — `install-report-날짜.txt`(요약), `install-log-날짜.txt`(상세).
5. **PATH 검증** — 설치된 도구가 명령어로 인식되는지 확인 후 출력.
6. **완료 안내 + 소요 시간 표시** + 다음 단계 가이드.

## 9. 안전한 버전(LTS)이란?

- 이 키트는 **베타(시험판)가 아닌 "안정 버전"** 만 설치합니다(winget 공식 소스 기준).
- **Node.js와 Java는 "LTS"** 로 설치합니다. **LTS = 오래 안정적으로 지원되는 검증된 버전**이라, 가장 안전합니다.
- **[6] 안전 업데이트** 도 같은 원리로 **LTS 라인 안에서만** 올립니다(갑자기 최신 실험판으로 튀지 않음).

## 10. 업데이트 / 제거 / 설치 확인

- **업데이트 [6]**: 설치된 도구를 안정(LTS) 버전으로 점검·업데이트. 실행하면 **업데이트 가능한 목록을 먼저 보여주고** "진행할까요?(y/n)" 를 물어봅니다.
- **제거 [7]**: `[1] 개별 제거`(번호 선택) 또는 `[2] 전체 제거`(이 키트로 깐 것 모두). 전체 제거는 **Y** 를 입력해야 진행됩니다.
- **설치 확인 [9]**: 도구별로 **[O](설치됨)/[X](없음)** 와 버전을 보여줍니다.

> 같은 PC에서 **다시 실행해도 안전**합니다(기본 "건너뜀"이라 기존 설정을 건드리지 않음).

## 11. 주요 명령어 (복사해서 사용)

설치가 끝나면 **새 터미널**(시작 메뉴 → Windows Terminal 또는 PowerShell)을 열고 확인하세요:

```
git --version
python --version
node --version
claude --version
```

Git 첫 사용 시 본인 정보 설정(한 번만):

```
git config --global user.name  "홍길동"
git config --global user.email "your@email.com"
```

> 설치 직후 **현재 창**에서는 명령어가 안 먹을 수 있습니다 → **새 터미널**을 열어 확인하세요(PATH 갱신 때문).

## 12. 문제·오류 대처법

| 증상 | 대처 |
|------|------|
| **"winget을 찾을 수 없습니다"** | Microsoft Store에서 **"앱 설치 관리자"** 검색·설치. 또는 설정 → 앱 → 선택적 기능 → 앱 설치 관리자 |
| **파란 경고창(SmartScreen)** | [추가 정보] → [실행] (정상입니다) |
| **창이 갑자기 닫힘** | 같은 폴더의 `install-log-날짜.txt` 마지막 줄에서 오류 확인 |
| **특정 도구만 설치 실패** | 화면 안내대로 **[8] 직접 다운로드**에서 그 도구만 받기 |
| **백신이 차단** | 해당 파일 허용 목록 추가 또는 실시간 보호 잠시 끄기 |
| **권한 때문에 안 깔림** | 파일 우클릭 → "관리자 권한으로 실행" |
| **한글이 깨져 보임** | Windows 기본 한국어 환경에서는 정상 표시됩니다. 영어 Windows라면 메뉴가 깨질 수 있습니다(향후 개선 예정) |
| **회사/학교 PC에서 막힘** | winget이 정책으로 차단된 경우가 있습니다 → [8] 직접 다운로드 이용 |

## 13. 파일·문서 위치

```
dev-one-click-setting-kit.bat   ← 메인 실행 파일 (이것만 있으면 됩니다)
README.md / README_EN.md        ← 간단 소개 (한/영)
GUIDE.md  / GUIDE_EN.md          ← 이 상세 설명서 (한/영)
GUIDE.pdf / GUIDE_EN.pdf         ← 설명서 PDF (md와 내용 동일)
CHANGELOG.md                    ← 버전별 변경 내역
LICENSE                         ← 라이선스 (MIT)
PRD/                            ← 초기 기획 문서 (참고용 / v1.0 설계 기록)
```

실행하면 **자동 생성**되는 파일(같은 폴더):
- `install-report-날짜.txt` — 성공/실패/건너뜀 요약 (강사·지원용)
- `install-log-날짜.txt` — 상세 기록 (오류 확인용)

> 마커 파일 `%LOCALAPPDATA%\devkit_intro_seen` — 환영 화면을 한 번 본 표시. 지우면 환영 화면이 다시 나옵니다.

## 14. 설치 후 워크플로우 (첫 바이브코딩까지)

1. 설치 완료 → **새 터미널**을 엽니다.
2. **Cursor**(또는 VS Code)를 실행합니다. *([8] 또는 [A]에서 Cursor 다운로드 페이지를 열어줍니다 — 직접 설치)*
3. **Claude에 로그인**합니다 (Claude Desktop, 또는 터미널에서 `claude` 명령).
4. 새 **폴더**를 하나 열고, 만들고 싶은 것을 **한국어로 그대로** 적어보세요.
   - 예) *"간단한 메모 앱 만들어줘"*
5. AI가 코드를 만들어 주면, 그대로 실행해 봅니다. 이게 바이브코딩입니다.

## 15. 자주 묻는 질문(FAQ)

**Q. 다시 실행해도 되나요?** — 네. 기본 "건너뜀"이라 안전합니다.
**Q. 특정 도구만 깔고 싶어요.** — **[5] 선택 설치**에서 번호로 고르세요.
**Q. 인터넷이 꼭 필요한가요?** — 네. 도구를 인터넷에서 받습니다.
**Q. 환영 화면을 다시 보고 싶어요.** — `%LOCALAPPDATA%\devkit_intro_seen` 파일을 삭제하세요.
**Q. Cursor·Claude Desktop은 왜 자동으로 안 깔리나요?** — 이들은 독립 프로그램이라, 사용자가 **직접** 설치하는 것이 맞습니다. [8]에서 다운로드 페이지를 열어줍니다.
